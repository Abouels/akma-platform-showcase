# Versioning Rules for AKMA

النسخة الحالية المرجعية التي لا يعاد تسميتها:
- `akma V2.02.01`

الصيغة الرسمية المستقبلية:
- `akma V<major>.<minor>.<patch>`

أمثلة:
- `akma V2.02.02`
- `akma V2.02.03`
- `akma V2.03.00`
- `akma V2.03.13`

## المعنى
- **Major**: مرحلة كبيرة أو تحول رئيسي في المنتج
- **Minor**: حزمة Features أو تطويرات واضحة داخل نفس المرحلة
- **Patch**: إصلاحات وتحسينات صغيرة أو متوسطة على نفس النسخة

## قاعدة العمل
- لا تعيد استخدام نفس الرقم لنسخة جديدة
- أي تعديل جديد بعد نسخة معتمدة = Patch جديد على الأقل
- لو الحزمة فيها تغييرات واضحة وكبيرة ضمن نفس المرحلة = ارفع Minor
